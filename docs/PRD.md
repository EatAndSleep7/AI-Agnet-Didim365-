# Product Requirements Document (PRD)
## 뱅킹 멀티 에이전트 시스템

---

## 1. 개요

### 1.1 제품명
**뱅킹 멀티 에이전트 시스템 (Banking Multi-Agent System)**

### 1.2 목적
은행 직원이 고객 관리, 규정 조회, 상품 추천, 실적 분석을 AI 어시스턴트를 통해 효율적으로 수행할 수 있도록 지원하는 내부 도구

### 1.3 배경
- 은행원의 업무 효율성 향상 필요
- Best Banker (베스트뱅커) 실적 관리의 복잡성
- 규정 정보의 빠른 검색 필요
- 개인화된 상품 추천의 어려움

---

## 2. 대상 사용자 (Stakeholders)

### 2.1 주 사용자: 은행원
- **역할**: 고객 응대, 상품 판매, 실적 관리
- **필요성**:
  - 고객 정보를 빠르게 요약하여 응대 시간 단축
  - 규정을 쉽게 조회하여 정확한 안내
  - 자신의 실적 현황을 즉시 파악
  - 고객 맞춤형 상품 추천 정보 획득

### 2.2 부 사용자: 관리자
- **역할**: 시스템 모니터링, 데이터 확인
- **필요성**:
  - Mock DB 조회를 통한 데이터 검증
  - 시스템 상태 모니터링

---

## 3. 핵심 기능 요구사항

### 3.1 Agent 1: 고객 정보 요약 에이전트
**목적**: 고객번호 입력 시 고객의 핵심 정보를 한 문장으로 요약

**기능**:
- 고객번호(CUST###) 기반 3개 테이블 조회
  - `customer_basic`: 기본정보, 잔액, 서비스 가입
  - `customer_profile`: 라이프스타일 추정, 소비 패턴
  - `customer_consultation`: 최근 상담 이력
- LLM으로 마케팅 관점의 한 문장 요약 생성
- 예: "마케팅 동의를 완료한 고액 수신 고객으로, 최근 여신 상품에 긍정적 반응을 보인 자가 주택 보유자"

**입력**: `고객번호` (TEXT)
**출력**: 한 문장 마케팅 요약 (자연스러운 한국어)

---

### 3.2 Agent 2: 베스트뱅커 규정 Q&A 에이전트
**목적**: 베스트뱅커 규정 및 정책 관련 질문에 답변

**기능**:
- Elasticsearch `bestbanker-2025` 인덱스 BM25 검색
- 검색된 규정 문서 상위 5개 반환
- 근거를 명시하여 답변 생성
- 규정집에 없는 내용은 "확인이 필요합니다" 안내

**입력**: 자유 형식 질문 (자연스러운 한국어)
**출력**: 규정 근거와 함께 한국어 답변

**예시 질문**:
- "베스트뱅커 10위 기준이 뭐야?"
- "가계여신 상품 추천 시 가점은?"

---

### 3.3 Agent 3: 베스트뱅커 현황 대시보드 에이전트
**목적**: 직원의 실적 현황을 분석하고 개선 방향 제시

**기능**:
- 직원번호(EMP###) 기반 3가지 분석
  1. 수신/여신/전자금융 3개 상품군별 현재 점수 조회
  2. 전체 직원 기준 10위 컷 점수 및 중앙값 산출
  3. 부족 상품군 자동 도출 (10위 컷과의 격차 최대)
- 비교표 형식의 현황 리포트 생성
- 개선을 위한 1줄 조언 제공

**입력**: `직원번호` (TEXT)
**출력**: 현황 리포트 + 부족 상품군 + 조언

**출력 예**:
```
📊 베스트뱅커 현황 리포트 (EMP001)
────────────────────────────────
상품군      | 내 점수 | 10위컷 | 격차
수신(예적금) | 142.5  | 180.0  | -37.5 ← 부족
여신(대출)   | 195.0  | 210.0  | -15.0
전자금융     | 88.0   | 95.0   | -7.0
────────────────────────────────
💡 가장 부족: 수신(예적금) — 37.5점 차이
```

---

### 3.4 Agent 4: 상품 추천 에이전트
**목적**: 고객 또는 직원 기반으로 최적의 상품을 추천하고 마케팅 문구 생성

**기능**:
- **경로 A (고객번호 있음)**
  - A-1) 고객 중심 추천: 해당 고객의 추천 점수 최고 상품 추천
  - A-2) 베스트뱅커 추진용: 직원의 부족 상품군에서 추천 점수 최고 상품 추천 (없으면 A-1로 폴백)
  - 고객 요약 → 마케팅 문구 자동 생성

- **경로 B (고객번호 없음)**
  - 직원의 부족 상품군 파악
  - 해당 상품군에서 추진 이력 고객 중 가장 많이 추천된 상품 선택
  - 해당 상품의 대상 고객 목록 및 요약 정보 제공

**입력**: `직원번호`, `고객번호` (선택)
**출력**: 추천 상품명 + 마케팅 문구 (경로 B: 대상 고객 목록)

**출력 예**:
```
추천 상품: 정기예금 플러스 (수신/예적금)
마케팅 문구: 현재 안정적 자산을 보유하신 고객님께, 높은 수익률의 정기예금 상품을 추천드립니다.
정기예금 플러스는 3개월~5년까지 다양한 만기를 제공하며...
```

---

### 3.5 Agent 5: 베스트뱅커 시뮬레이션 에이전트
**목적**: 특정 상품을 추진했을 때의 예상 점수 변화를 시뮬레이션

**기능**:
- 직원번호 + 상품코드 입력 시
  - 현재 점수 → 추진 후 예상 점수 계산
  - 합계 점수 변화
  - 전체 순위 변화 계산
- 해당 상품의 규정집 근거 자동 검색 및 인용
- 유의사항 안내

**입력**: `직원번호`, `상품코드` (TEXT)
**출력**: 시뮬레이션 리포트 + 규정 근거 + 유의사항

**출력 예**:
```
🎯 베스트뱅커 시뮬레이션 (EMP001 / DEP101)
────────────────────────────────
상품: 정기예금 플러스
가점: +12.5점

수신 점수 | 142.5 → 155.0 (+12.5)
합계 점수 | 425.5 → 438.0 (+12.5)
순위      | 12위 → 9위 (▲3위)

📋 규정: 정기예금 가입 건당 12.5점 부여, 동일 고객 월 1회 한정...

⚠️ 유의사항: 규정집의 상한선 확인 필수
```

## 4. 유저 스토리

### 4.1 고객 상담 시나리오
```
은행원: "어떤 고객을 조회하고 싶으신가요?"
채팅: "CUST012 고객을 알려줘"
에이전트: "마케팅 동의를 완료한 고객으로, 최근 여신 상품에 긍정적 반응을 보인 자가 주택 보유자입니다."
은행원: "좋아, 이 고객에게 추천할 상품이 뭐야?"
에이전트 : 고객 중심인가요, 직원 중심인가요 ?
은행원 : 직원 중심
에이전트: "정기예금 플러스를 추천합니다. 마케팅 문구: ..."
은행원: (고객에게 마케팅 문구 사용하여 상품 설명)
```

### 4.2 실적 관리 시나리오
```
은행원: "EMP001 직원인데 내 현황을 봐줘"
에이전트: (현황 대시보시드 제시)
은행원: "수신이 부족하다고 했는데, 수신 상품을 하나 추진하면 순위가 어떻게 돼?"
에이전트: (시뮬레이션 결과 제시, 규정 근거 포함)
은행원: "수신상품 추천할만한 고객 찾아줘."
에이전트: (수신상품 추천할만한 고객 목록 제시)
```

### 4.3 규정 확인 시나리오

```
에이전트: (규정집 검색 후 최신 가점 정보 제시)
```
